# Zing! OS 배포 및 운영 가이드 🚀

Zing! OS를 실제 서비스로 운영하기 위한 최종 단계입니다.

## 1. Supabase 데이터베이스 설정 (필수)
Supabase SQL Editor에 접속하여 아래 쿼리를 실행해 테이블을 생성해주세요.

```sql
-- 1. Ideas 테이블 생성
create table ideas (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  title text not null,
  content text,
  theme text default 'default',
  folder text,
  author_handle text,
  author_avatar text,
  image_url text,
  tag_label text,
  tag_color text,
  parent_id bigint references ideas(id),
  likes int default 0
);

-- 2. Comments 테이블 생성
create table comments (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  idea_id bigint references ideas(id) on delete cascade,
  user_handle text,
  content text,
  user_avatar text
);

-- 3. 권한 설정 (모든 사용자가 읽고 쓸 수 있도록 - 테스트용)
alter table ideas enable row level security;
create policy "Anyone can view ideas" on ideas for select using (true);
create policy "Authenticated users can create ideas" on ideas for insert with check (auth.role() = 'authenticated');

alter table comments enable row level security;
create policy "Anyone can view comments" on comments for select using (true);
create policy "Authenticated users can create comments" on comments for insert with check (auth.role() = 'authenticated');
```

## 2. Vercel에 배포하기
1. GitHub 저장소 생성 후 현재 코드를 `push` 합니다.
2. [Vercel Dashboard](https://vercel.com/dashboard)에서 **Add New > Project** 클릭.
3. GitHub 저장소 연결.
4. **Environment Variables** 설정:
   - `VITE_SUPABASE_URL`: 당신의 Supabase URL
   - `VITE_SUPABASE_ANON_KEY`: 당신의 Supabase Anon Key
5. **Deploy** 버튼 클릭!

## 3. 마케팅 준비 (SEO)
- `index.html`에 이미 기본적인 SEO 설정을 완료했습니다.
- `public/og-image.png` 위치에 서비스의 메인 스크린샷이나 로고 이미지를 넣어주세요. 링크 공유 시 해당 이미지가 노출됩니다.

## 4. 관리자 계정 생성
- 서비스 배포 후 `admin@zing.io` 이메일로 가입하세요. 
- Supabase 대시보드에서 해당 유저의 `Email Confirmed` 상태를 확인하면 바로 관리자 도구를 사용할 수 있습니다.
